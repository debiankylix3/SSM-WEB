Arduino Mightycore 1284P Plug & Play

Main Board Topology Documentations

Function Overview

The board offers high connectivity thanks to the numerous number of installed ports; it has I2C, Serial (UART), SPI, Wiegand, 3.5mm Jack, Ethernet, and the special ports used to access PORT A, PORT B, PORT C and PORT D of the microprocessor (for more detail check the pinout diagram). The high connectivity makes the board suitable for almost every entry-level and education scenario.

The well-designed microstrip on the board that connects external modules to the board makes your professional embedded system’s wiring hassle-free. Powered by the powerful ATMega 1284p chip, this board can overcome almost every challenge in your embedded system projects. The board’s high robustness means it can withstand the outdoor operating scenario (while put in a case).

The following figure and the table below describe the key components, interfaces and controls of the Arduino MightyCore 1284P Plug & Play board.

No. Key Component Description
1. ATmega 1284P Microcontroller with AVR as its core. For more information, see ATMega 1284P Datasheet
2. Buzzer Generate a pure frequency tone based on the signal it received
3. DS3231SN RTC IC with I2C interface and commonly employed for precise timekeeping applications
4. ENC28J60 Ethernet controller chip operating at 10 Mbps through SPI
5. Jack 3.5mm port Connect the board with audio devices (active speaker, headphone)
6. FTDI port To connect with FTDI modules for debugging purposes which uses usb interface
7. Serial port Connect with various devices that support serial interface
8. Power port 5V External power if the board is not connected to device
9. I2C port To connect with various devices that support I2C interface
10. SD Card reader A device to read and write data to SD or memory card.
11. Ethernet port A port that allows the board to communicate through the plugged ethernet cable.
12. Relay Port Special port to connect relay with the board
13. Keypad Port A port that allows the connection of the keypad, a keypad can be programmed to give inputs
14. Wiegand Port A port that allows the board to connect to a device that use wiegand protocol (i.e. patrol guard’s RFID scanner)
15. DF Player Port A port that allows the connection of a DF Player. DF Player is an external module used to read / write data from a microSD card
16. Button Port A port that allows buttons to be connected to the board. Buttons can be programmed and used for numerous purpose (i.e. turn on / off LED)
17. TCP/IP Port A TCP/IP port on a board is a designated endpoint for communication within a network using the TCP/IP protocol
18. SPI port Connect device with SPI interface such as ethernet
19. LED port Connect a LED to the board
